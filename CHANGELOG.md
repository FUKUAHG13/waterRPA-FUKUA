# 变更记录

## v1.0.11 - 2026-07-13

- 为步骤和跳转增加稳定 ID，拖动、插入、复制和删除后仍按原目标跳转；删除被引用步骤会先提示。
- 恢复主列表拖动插入位置预览，并把录制升级为双击、组合键、滚轮合并和真实拖拽时长语义。
- 新增启动/等待/激活/关闭窗口，以及 UI Automation 控件点击、写值和读值动作。
- 新增 Windows DPAPI 凭据库与秘密文本输入；普通文本和秘密文本均不进入日志，也不再占用剪贴板。
- 全量导出遇到缺失图片时停止生成残缺包；只读安装目录自动改用本地用户数据目录。
- 设置增加简易/高级视图，脚本检查改为可定位问题列表，步骤字段编辑纳入撤销历史。
- 发布门新增 10 轮相对性能基线，覆盖 5000 步核心路径、OpenCV、原生调度和分阶段启动。

## v1.0.10 - 2026-07-13

- 方案兼容改为有边界的主版本策略：同一主版本继续向前迁移，大版本可提高最低方案版本，不再承诺无限历史兼容。
- 原始方案签名改为迁移前验证；兼容迁移前保留旧自动备份，避免新增默认字段被误判为损坏。
- 旧程序遇到过新方案或过新自动备份时进入只读保护，禁止自动/退出保存覆盖原文件，并给出明确处理提示。
- 日志“内容...”菜单支持连续勾选和连续恢复预设，只有点击外部或按 Esc 才关闭。
- 界面倍率统一为文字按钮计算内容基准宽度，并按最终字体提示兜底，修复 180% 下主工具栏文字末字被裁切。
- onedir/onefile 共用经过冻结烟雾验证的运行库删减策略，移除 OpenCV FFmpeg、Qt Quick/QML/PDF/虚拟键盘链、Pillow 可选扩展、Tcl/Tk 辅助链及非中文 Qt 翻译；实验构建的 `_internal` 净减 70.52 MiB。
- 离线自检新增无副作用的 PyAutoGUI 输入与 1×1 截图后端检查，防止可选依赖裁剪后只剩“可导入但不可使用”的隐性故障；本版本仍未生成正式发布包。

## v1.0.9 - 2026-07-13

- 日志系统新增可保存的“自定义”模式和“内容...”菜单，可分别控制运行进度、步骤结果、流程分支、动作、识别、坐标、参数、耗时、底层后端、画面唤醒/缩放记忆、界面事件和本地时间。
- 简易、详细、完全三个预设改由统一 `LogPolicy` 定义；自定义过滤发生在异步日志入队前，关闭某类内容时不再产生对应的文件或界面刷新开销。
- 错误、急停和严重警告采用不可关闭的强制类别；旧的未分类调用继续按步骤结果输出，避免升级后静默丢日志。
- 配置架构升级至版本 2，旧方案按原日志级别自动迁移；自定义项目可随原子备份、普通方案和全量方案导入导出。
- 详细/完全及启用时间项的自定义界面日志统一使用事件发生时的本地毫秒时间；完全/自定义耗时项可输出完整步骤阶段增量。
- 资源监测新增可保存的 0.5/1/2/3/5 秒、跟随省电模式和关闭选项，状态栏改为系统整体与本进程 CPU，继续使用非阻塞 Qt 定时采样。

## v1.0.8 - 2026-07-13

- 修复同一图片仅因缩放上下限或步长变化而生成多条缩放记忆：记忆改按图片身份归并，范围变化只过滤当前无效倍率；图片文件实际被替换时安全清空旧历史。
- 新增默认启用的“画面变化唤醒”和稳妥/均衡/灵敏三档，只缩短自适应降频产生的额外等待，不改变基础识别频率、失败分支或完整搜索兜底。
- 画面监听优先使用 DXGI 脏矩形作为低开销通知，再由分块灰度指纹确认真实变化；DXGI 不可用时自动回退原生低分辨率 GDI，最后回退 Python 截图。
- 画面变化确认后先用新截图检查已有常用倍率，并轮换少量剩余倍率；快速探测未命中时立即执行原来的完整范围搜索，“全部匹配”和同点上限继续走完整语义。
- C++ 核心升级至 API 11200：保留全部旧入口，新增指定倍率专搜、低分辨率画面指纹和 DXGI 变化通知能力；仍为 x64 `/MT` C++17/Windows 10 目标且无需另装运行库。
- 原生真实屏幕烟雾新增“指定倍率未命中不得暗中回退”、低分辨率指纹和 DXGI 实际画面变化验证，并补充唤醒配置、回退、停止和倍率轮换的定向测试。

## v1.0.7 - 2026-07-13

- 设置页首次打开时显示一次快速反应提示，解释间隔、识别频率、自适应降频与 CPU 占用的关系；提示状态独立于方案导入导出。
- 两个避让坐标新增直接取点，复用原生鼠标 Hook，不隐藏设置窗口且支持虚拟桌面负坐标。
- 避让后操作新增仅移动、最终点左键单击和最终点右键单击；二段避让只在坐标2额外点击一次，默认行为保持仅移动。
- 项目守则明确区分日常定向测试与打包发布质量门：未要求打包时不再运行数百项完整测试和发行态烟雾。
- 本版本当前仅更新源码，尚未生成发行包。

## v1.0.6 - 2026-07-12

- “快速一个”从只记住上次倍率升级为会话内多倍率学习：按命中频率、近期程度和匹配质量动态排序常用倍率，未命中仍复用同一截图完整回退。
- 设置新增稳妥/均衡/积极三档自动策略、最多 12 个手动优先倍率及可选自定义学习数量与历史容量；学习记录只驻留内存，不跨重启保存。
- C++ 识别核心升级至 API 10900，新增多倍率优先列表入口与能力位，同时保留 API 10800 单倍率入口、旧入口和 OpenCV/Python 回退。
- 图片点击的详细/完全日志显示实际命中倍率；设置页实时展示每张图片的当前优先倍率、命中历史数和算法计算的历史上限，完全日志只在优先集合或容量变化时记录。
- 启动改为先绘制轻量工作区准备窗口，再异步导入完整界面；OpenCV、NumPy、PyAutoGUI 和 UI Automation 分别延迟到运行后端初始化或首次使用。
- 新增源码与冻结 EXE 启动烟雾，验证首帧、工作区可见和运行后端就绪的阶段顺序与宽松性能上限。
- 修正原生多核选择框宽度、连续删除后的橙红悬停状态，并将删除后的延迟刷新绑定到主窗口生命周期。
- 默认仍只构建完整版多文件版；完整版单文件版继续同源维护，只有明确要求时才构建。

## v1.0.5 - 2026-07-12

- C++ 识别核心升级至 API 10800：每次识别使用一个有界任务池，统一调度缩放档位、区域和行块，替代逐缩放反复创建线程。
- 设置新增原生多核的自动、单线程和强制模式；自动模式按总计算量启用最多 8 个线程，不设置核心亲和性。
- “快速一个”可优先检查同一目标上次成功的缩放；预测未命中时在同一截图上继续完整范围，“全部匹配”始终完整搜索。
- 保留旧 DLL 调用入口和 OpenCV/Python 回退，不新增机器学习或大型运行库；真实窗口烟雾覆盖两种并行模式和缩放回退。
- 默认构建改为完整版多文件版；单文件版继续同源维护，但只有明确要求时才构建，精简版保持终止。

## v1.0.4 - 2026-07-12

- 统一响应式设置行的行内/组间水平间距、控件垂直中心和换行高度回传；窗口变宽时内容高度、滚动范围与滚动条滑块会同步变化。
- 修复小齿轮中“取/图/区”等紧凑按钮裁字，点位序列改为启用后再展开详细设置。
- 设置页未聚焦的下拉框不再被滚轮误改；映射目标状态移到每行最右侧并在宽窗口显示完整名称。
- 主界面调试标题与操作按钮统一放到右侧；播放倍率名称改为“播放倍速”。
- 删除步骤时显式销毁旧行控件并刷新悬停提示，避免连续删除时残留旧的鼠标状态。
- 条件断点移入默认折叠的“高级调试”，新增常用暂停条件预设，同时保留原有安全表达式。
- 恢复同源完整版单文件构建；默认发布同时生成 onedir 与 onefile，精简版继续停更。

## v1.0.3 - 2026-07-12

- OpenCV 线程预算统一限制为最多 2 个，并由引擎、离线诊断和视觉基准共同报告与验证。
- UIA 目标选择优先匹配明确绑定的子窗口，否则按深度、面积和动作能力排序；离屏控件被排除。
- 新增安全条件断点、断点条件确定赋值检查和求值错误可解释暂停，调试详情不写入持久轨迹。
- 发布目录新增最终用户说明，发布信息独立记录 EXE、原生核心与 UIA 辅助 DLL 的哈希和真实签名状态。
- 一键构建新增流式便携 ZIP、逐成员清单复核及 ZIP/目录清单外部 SHA256SUMS。
- C++ 识别核心升级至 API 10700，公开 ABI 位数、结构体尺寸和 x64 `/MT` C++17/Win10 编译标志。
- 原生构建增加 PE 导入表检查，禁止依赖 VCRUNTIME/MSVCP/UCRT 动态运行库。
- 发布门新增全部 EXE/DLL/PYD 的 AMD64 与依赖闭包审计，并随包保存 `RUNTIME_CLOSURE.json`。
- 原生烟雾测试扩展到灰度、彩色、多尺度、全部匹配、多区域排除及 OpenCV 坐标对照，并可由打包后 EXE 运行。
- 质量门扩展到 152 项测试；继续仅发布 Windows 10 1809+/Windows 11 x64 完整多文件版。

## v1.0.2 - 2026-07-12

- 后台单次左键操作新增有界 UI Automation Invoke/Selection/Toggle/DefaultAction，并保留 PostMessage 回退。
- UIA 超时加入取消令牌；动作结果未知时禁止重复发送窗口消息，真实 Win32 控件烟雾测试确认不会抢前台。
- 新增步骤前断点、F8 切换、暂停、继续、单步越过和仅在暂停界面显示的运行变量。
- 新增“设置变量”“判断表达式”以及 AST 白名单解释器和确定赋值流程检查，不执行任意 Python 代码。
- C++ 识别核心升级至 API 10600：稳定能力掩码、预算返回码和连续失败三次的运行级熔断。
- 新增 CycloneDX SBOM、第三方声明、许可证汇总、真实 Authenticode 状态和设置页后台完整性校验。
- 发布包同时生成内部与外部逐文件哈希；额外可执行文件会使校验失败，且明确内部哈希不等于作者签名。
- 设置页顶部操作区支持响应式换行；质量门扩展到 137 项测试、UIA 烟雾与禁止裸异常捕获。
- 阶段五至八完成并细化维护边界；继续只发布 Windows 10 1809+/Windows 11 x64 完整多文件版。

## v1.0.1 - 2026-07-12

- 统一设置行距并压缩固定选项框宽度；界面倍率现在同步缩放字体和控件。
- 修复倍率输入框按 Enter 误触导出，热键录入实时显示当前组合。
- 新增单步执行、有界性能遥测、离线诊断导出、脚本流程检查和不含敏感步骤内容的运行轨迹。
- C++ 识别核心升级至 API 10500：每区域单次截图/积分图、模板 LRU、预算前置回退和线程上限。
- MSS 失败后本轮熔断；OpenCV 基础模板、同区域条件截图和重复状态刷新不再重复处理。
- 新增后台目标窗口/控件检查器、子控件枚举和完整性级别诊断。
- 新增视觉基准、真实屏幕正向烟雾、内存耐久、构建记录与一键 onedir 发布脚本。
- 支持范围保持 Windows 10 1809+ x64 与 Windows 11 x64；仅发布完整多文件版。
