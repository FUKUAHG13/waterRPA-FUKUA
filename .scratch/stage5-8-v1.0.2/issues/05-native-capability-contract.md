# 原生核心能力契约与运行级熔断

## Status

Completed

## Comments

- DLL API 升级到 10600，并通过位掩码声明截图、区域、缩放、灰度、全匹配、缓存、计时和预算能力。
- 工作预算回退优先依据稳定返回码 `-2`；旧 DLL 的英文错误文本只保留为兼容路径。
- 普通原生失败连续三次后仅在本次运行停用 DLL，下次启动脚本重新探测，OpenCV 回退始终保留。

## Acceptance

- 构建记录、离线自检和诊断导出均报告 API、能力位和最近结果。
- 确定性的工作预算拒绝按识别签名缓存，不重复调用 DLL，也不计入普通失败熔断。
- 连续普通失败触发运行级熔断，成功或权威空结果会清零连续失败计数。
- 正式发布必须携带 API 10600，并通过原生正向烟雾、基准和回退测试。
