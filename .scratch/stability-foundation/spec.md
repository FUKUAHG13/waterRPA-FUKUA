# 稳定地基与质量建设

## 目标

- 在不改变现有用户配置、脚本行为和多文件发布方式的前提下，降低主窗口与执行引擎的耦合。
- 建立版本化配置迁移、明确的运行生命周期、可复用验证器和故障恢复基础设施。
- 扩充自动化测试，覆盖配置迁移、调度边界、异常恢复、资源释放和模块边界。
- 最终生成仅支持 Windows 10 1809+ x64 与 Windows 11 x64 的完整版多文件包。

## 不可破坏约束

- 原生识别返回 `None` 时回退，返回空列表时不得重复 OpenCV 搜索。
- 每次运行必须由同一个 `run_id` 完成 `reserve_run()` / `finish_run()` 配对。
- 脚本坐标使用 Windows 虚拟桌面物理像素，Qt 覆盖层负责坐标转换。
- 配置先写原子 JSON 备份，再更新 `QSettings`，并继续读取旧 `waterRPA` 格式。
- 保留 Python/OpenCV 回退路径，原生 DLL 不是强制依赖。
- 只维护完整版多文件构建。
