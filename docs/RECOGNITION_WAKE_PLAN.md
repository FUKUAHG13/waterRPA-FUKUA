# 图片识别空闲唤醒实施方案

状态：v1.0.8 已完成三个阶段并进入发布验证。

## 当前问题

连续未命中三次后，自适应降频会逐步增加等待，最高额外 0.8 秒。v1.0.8 在不改变完整搜索语义的
前提下监听等待期间的画面变化，并在变化确认后用新截图做有界快速探测。

## 实施边界

- 普通一次性识别、失败跳转、全部匹配和“直到条件成立”的既有语义不能被静默改变。
- 优先倍率探测不能永久取代完整倍率搜索；任何模式都必须有有界的完整搜索兜底。
- 不并发运行两套截图/匹配线程，不以重复抢屏换取表面延迟。
- 不引入机器学习或新大型运行库，新增功能必须可关闭。

## 阶段一：测量与低风险唤醒（已完成）

1. 分开记录基础等待、自适应额外等待、优先倍率、完整回退、变化比较和唤醒次数。
2. 设置新增默认启用的“画面变化唤醒”，只作用于自适应增加的等待，不改变一次识别的完整回退。
3. 优先使用本步骤识别区域，其次使用全局区域；多区域分别保存轻量基准。
4. DXGI 脏矩形只作为廉价通知，随后使用缩小图加分块差异确认，避免小目标被整屏平均值稀释。
5. 动画导致持续变化时最多退化为关闭自适应降频，不能漏检或延长原有最大等待。

## 阶段二：常用倍率快速探测（已完成）

1. 为原生 API 增加“仅检查给定倍率、不自动完整回退”的可选入口，旧入口继续保留。
2. API 11000 的 `wrpa_find_template_ex3` 只匹配明确给出的倍率；API 10900 及更早入口保持原行为。
3. 变化确认后有常用倍率才运行专搜；命中结果由紧接着的正常调用消费，避免重复截图和匹配。
4. 没有学习记录、全部匹配或同点点击上限时不运行专搜，直接进入当前完整搜索语义。

## 阶段三：有界轮换剩余倍率（已完成）

每次快速探测除常用倍率外，按灵敏度带入 1 至 3 个剩余倍率并在后续等待中轮换。它只增加早命中
机会，不承担失败判断；专搜未命中后同一次正常调度立即完整搜索全部倍率，因此不存在等待多轮才
覆盖完整范围的问题。

## 原生回退层

1. API 11200 优先使用 DXGI Desktop Duplication 查询脏矩形，静止时不复制屏幕。
2. 动态通知按灵敏度节流，并由低分辨率 GDI 分块指纹确认，避免动画通知直接关闭降频。
3. DXGI 初始化、驱动、远程桌面或显示切换失败时回退低分辨率 GDI；仍失败才使用 Python/MSS。
4. 所有路径都可由“画面变化唤醒”开关整体关闭，关闭后恢复原自适应等待。

## 验收场景

- 目标连续出现与空闲 1、2、5 秒后首次出现的响应延迟。
- 小目标出现在大区域、缓慢淡入、记分板持续变化、背景动画和完全静止画面。
- 常用倍率命中、未知倍率命中、优先倍率全部失败、全部匹配和 DLL 关闭回退。
- 开启唤醒后的 CPU、截图次数、P50/P95、漏检率和停止响应，必须同时优于或不劣于关闭自适应降频。
