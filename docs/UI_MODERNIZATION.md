# fukuaRPA 界面现代化方案

## 目标与边界

- 不增加第三方 UI 框架、图片资源包或字体，控制安装体积与兼容风险。
- 保留现有 Qt 控件、信号和配置字段，视觉改造不改变脚本执行行为。
- 支持 Windows 10 1809+ / Windows 11 x64，并保留用户自定义界面倍率。
- 优先改善信息层级、可读性、状态反馈和小窗口适配，不追求高成本动画。

## 第一阶段：统一设计基础（已完成）

- 新增集中式主题，统一字体、颜色、边框、间距、圆角、悬停、按下和禁用状态。
- 为主要、次要、危险、成功、图标按钮建立语义样式。
- 使用 Qt 自带系统图标，不新增图片资源或运行依赖。

## 第二阶段：主工作区重排（已完成）

- 建立清晰的方案栏、命令栏、脚本步骤区、运行区、日志区和底部状态栏。
- 增加品牌与步骤数量提示、空列表状态、选中步骤高亮和更清楚的拖放插入线。
- 固定关键控件的最小尺寸，避免内容变化造成工具栏抖动或文字截断。

## 第三阶段：设置与辅助窗口（已完成）

- 全局设置和步骤设置使用统一的分组、折叠区、帮助按钮与底部操作区。
- 拆分过长的横向设置行，保证全局设置在 720x520 的最小尺寸下不出现横向滚动。
- 设置项使用响应式流式布局：窗口变宽时自动合并到同一行，变窄时只在完整控件组之间换行。
- 复杂设置统一显示 `?` 帮助按钮，并同时支持悬停与点击；图标和纯操作按钮只保留简短悬浮说明。
- 文字按钮和下拉框按当前字体内容计算宽度，不再用容易在高 DPI 或界面放大时裁字的固定宽度。
- 统一坐标选择、录制、预览、运行提示和点击提示浮层的视觉语言。

## 第四阶段：状态与可访问性（已完成）

- 启动、停止、删除、录制等操作使用稳定且一致的颜色语义。
- 保留明亮日志背景，确保现有详细日志中的蓝、红、绿文字仍可辨识。
- 给图标按钮保留工具提示和可访问名称，设置窗口与主窗口保持同级关系。
- 使用统一的 fukuaRPA 品牌图标替代默认 EXE 图标，并覆盖窗口标题栏、任务栏和构建资源。

## 第五阶段：验证与交付（已完成）

- 增加 UI 回归测试，覆盖主题、控件语义、最小尺寸、横向溢出、响应式换行、文字完整性和界面倍率。
- 设置页顶部发布操作同样使用响应式布局，在高倍率和窄窗口下按完整按钮边界换行。
- 使用源码窗口和最终 PyInstaller 完整版冻结包分别进行真实 Windows 截图检查。
- 同一套源码生成完整多文件版与完整单文件版；精简版不再生成。

维护者可运行 `python scripts/capture_ui_review.py` 生成主窗口、设置页和步骤设置的固定尺寸审查图；
最终发布仍需使用 Windows 原生 Qt 后端检查真实中文字体回退。

## 后续可选增强

- 为超长步骤设置增加页内分类导航，减少连续纵向滚动。
- 在不改变配置结构的前提下，逐步补充键盘焦点样式和高对比度主题测试。
- 若未来加入动画，只允许短时、可关闭的状态过渡，不应用于高频日志或步骤刷新。
