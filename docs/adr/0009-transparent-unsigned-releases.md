# ADR 0009：透明的未签名发布与双层完整性校验

- 状态：已接受
- 日期：2026-07-12

## 决策

每个多文件发布生成源码指纹与构建记录、CycloneDX SBOM、第三方声明、许可证汇总、真实
Authenticode 状态、包内 `PAYLOAD_HASHES.json` 和包外逐文件 SHA-256 清单。设置页可在线程中
执行包内校验；清单外日志或配置只警告，清单外 EXE/DLL/PYD/PY/PYC/ZIP 视为失败。

没有作者证书时明确记录 `NotSigned`，不创建测试证书冒充发布身份。程序不在启动时联网，
也不自动检查更新。

## 原因

内部哈希能发现意外损坏，但不能证明发布者身份，因为攻击者可以同时替换文件和内部清单。
外部清单便于独立核对；真正的身份认证仍必须依赖受信任代码签名证书。

## 后果

正式签名属于证书可用后的独立发布步骤。当前 UI 必须明确说明“完整性校验不等于数字签名”，
构建脚本必须在所有发布文件生成后再次检查包内与包外清单。
