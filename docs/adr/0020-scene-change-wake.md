# ADR 0020：画面变化唤醒与有界倍率专搜

## 状态

已接受，v1.0.8 实施。

## 背景

自适应降频会在同一图片连续未命中后增加最多 0.8 秒等待。它能降低空转 CPU，但目标在等待期间
出现时，空闲后的第一次响应会明显慢于连续出现。单纯提高截图频率会重新增加 CPU；单纯只搜记忆
倍率则会改变失败和完整缩放搜索语义。

## 决策

1. 新功能只观察并缩短自适应增加的等待，基础识别频率仍完整保留。
2. DLL API 11200 优先用 DXGI Desktop Duplication 的脏矩形作低成本通知；通知本身不算变化成功，
   必须再用缩小灰度图的局部/整体分块差异确认。DXGI 失败回退原生 GDI 指纹，再失败回退 Python。
3. 变化确认后，只有已有手动或学习优先倍率时才调用 API 11000 的指定倍率专搜，并附带一个有界
   轮换批次。命中结果只缓存给紧接着的正常搜索调用。
4. 专搜未命中立即执行原来的完整搜索。全部匹配、同点点击上限及没有优先倍率的步骤不做专搜。
5. 缩放记忆身份不再包含缩放范围；范围变化只过滤历史交集。文件代次变化会清空同一路径旧历史。

## 后果

- 静止等待时 DXGI 轮询不复制屏幕；动态桌面只按灵敏度节流指纹确认。
- 首次初始化显卡复制接口可能有一次性开销；失败后自动回退，不阻止脚本执行。
- 动画仍可能构成真实画面变化并提前完整识别，但不会造成漏检或超过原有最大等待。
- 不增加机器学习模型或第三方运行库；D3D11、DXGI 和 GDI 均由受支持的 Windows 提供。
