# ADR 0004：显式选窗与原生热键录入

- 状态：已接受
- 日期：2026-07-11

## 决策

按键映射的点击坐标与后台目标窗口身份分开采集。坐标由“取点”设置；只有选择
“后台窗口点击(实验)”后，“选择窗口”才可用，用户随后必须亲自单击目标程序窗口。
绑定使用该次选窗得到的根窗口和原有点击坐标解析目标控件，不再根据点击坐标猜测当前最上层程序。

热键录入使用 `WH_KEYBOARD_LL`，由 Hook 的按下/抬起事件维护 Ctrl、Alt、Shift、Win
状态；不在回调中依赖可能尚未更新的 `GetAsyncKeyState`。Hook 安装失败时保留 Qt 按键事件
兼容路径。

## 原因

坐标上方的最前端窗口可能是设置页、预览层或其他程序，自动猜测不能表达用户意图。
Qt 普通按键事件又可能在 Alt 菜单键、系统键和快捷键分发过程中丢失组合信息。

## 后果

选窗时目标程序必须未最小化，映射坐标必须位于所选窗口的可点击客户区。后台发送失败只报告
失败，绝不改用真实鼠标点击。录键弹窗会在录入期间拦截按键，完成或取消后必须停止 Hook。
