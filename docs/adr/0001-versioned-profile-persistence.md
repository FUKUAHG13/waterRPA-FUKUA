# ADR 0001：版本化方案与签名备份

- 状态：已接受
- 日期：2026-07-11

## 决策

方案使用独立于产品版本的 `_schema_version`，只允许通过显式迁移向前升级。保存时先写带
SHA-256 签名的原子 JSON 备份，再写 `QSettings` 及相同签名；启动时检查签名握手，并按
“主配置、当前备份、有限历史备份、默认方案”恢复。

兼容窗口以产品主版本为边界：同一主版本内继续迁移旧方案；只有大版本可以提高最低支持
方案版本。遇到过旧或由更新程序写入的方案时，程序只读拒绝并禁止自动保存，原配置和自动
备份不得被默认方案覆盖。执行兼容迁移前保留升级前备份。

## 原因

产品版本无法可靠表达配置兼容性。单份备份也无法覆盖“备份写完但 INI 未写完”或备份本身
损坏的情况。未知字段必须保留，未来版本配置不得被旧程序静默降级。

## 后果

新增持久字段时必须同时更新默认值和迁移。原始签名必须在迁移前验证，迁移后再比较主配置
与备份的逻辑内容。修改保存顺序、签名算法、兼容下限或恢复优先级前必须增加故障注入测试。
