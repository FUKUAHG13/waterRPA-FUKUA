# ADR 0008：断点调试与受限表达式

- 状态：已接受
- 日期：2026-07-12

## 决策

执行引擎使用条件变量实现步骤前断点、暂停下一步、继续和单步越过。停止会唤醒暂停线程，
调试暂停时长从全局运行时限中扣除。断点状态随步骤保存，单独执行某一步时强制忽略断点。

运行变量只存在于一次脚本运行中。“设置变量”和“判断表达式”使用 AST 白名单解释器，
禁止 `eval`、`exec`、调用、属性、下标、集合和导入，并限制源码长度、节点数、数值、文本、
幂运算和变量数量。流程检查器执行确定赋值分析，提示某条路径上可能尚未设置的变量。

## 原因

真正的断点必须协调 Worker 与 UI，不能轮询 Qt 控件。直接执行 Python 表达式会把文件、进程、
网络和任意代码能力暴露给脚本配置，不适合作为面向普通用户的自动化条件。

## 后果

表达式不是 Python 的兼容子集，只支持文档列出的算术、比较和布尔操作。变量快照只显示在
当前暂停界面，不写入运行轨迹、日志或诊断导出。
