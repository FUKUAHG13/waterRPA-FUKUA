# ADR 0005：有界性能遥测与原生识别复用

- 状态：已接受
- 日期：2026-07-12

## 决策

运行时对截图、裁剪、模板预处理、DLL/OpenCV 匹配、点击、状态和日志刷新进行低开销计时。
每个指标只保存最近 256 个样本，同时累计全程次数和总耗时。DLL API 10500 在一次调用中
每个识别区域只截图一次、只构建一次积分图，并使用最多 64 项或 256 MB 的模板 LRU。

超过 DLL 工作预算是确定性回退：C++ 在截图前返回，Python 在本次脚本运行中缓存该结果，
后续直接走 OpenCV。DLL 返回空列表仍表示权威的“未命中”，不得触发重复搜索。

## 原因

旧实现会按“缩放档位 × 区域”重复截图和建立积分图，且每轮重新转换模板；只看一次最快值
无法定位这些成本。固定上限可在无限循环中保持内存稳定，P50/P95 比单次最小值更有代表性。

## 后果

发布质量门包含 1080p/2K/4K、灰度/彩色、多区域、多缩放基准和真实屏幕正向烟雾测试。
详细/完全日志的刷新语义保持不变；用户仍可通过切换简易日志或关闭界面日志控制其开销。
