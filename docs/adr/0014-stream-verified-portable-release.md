# ADR 0014：流式复核的便携发行归档

- 状态：已接受
- 日期：2026-07-12

## 决策

一键发布同时保留可直接运行的 onedir 目录并生成以构建名为唯一根的便携 ZIP。ZIP 采用流式读写，随后对
外部逐文件清单复核路径、数量、大小、SHA-256 与 CRC；再为 ZIP 和目录清单生成并复核外部 SHA256SUMS。

## 原因

用户需要单个文件传输完整多文件程序，但只生成 ZIP 并不能证明归档内容等于已验证目录。逐成员复核可以
发现路径穿越、大小写冲突、重复/加密/清单外成员、截断和内容替换，同时避免把数百 MB 发布物整体载入内存。

## 后果

归档必须在全部包内物料生成并验证后创建，且不能写入待归档目录内部。哈希只能证明文件与所给清单一致，
不能证明作者身份；没有受信任代码签名证书时仍需明确标记未签名，不得用测试证书代替。
