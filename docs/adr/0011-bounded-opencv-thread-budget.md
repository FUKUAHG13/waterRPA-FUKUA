# ADR 0011：有界 OpenCV 线程预算

- 状态：已接受
- 日期：2026-07-12

## 决策

所有 OpenCV 路径通过 `opencv_runtime.configure_opencv_threads()` 设置进程级线程数，默认和上限均为 2，
单核心机器自动降为 1。执行引擎、离线诊断、视觉基准与原生对照烟雾必须调用同一入口并报告实际值。

## 原因

在高核心数处理器上让 OpenCV 使用全部逻辑核心会显著抬高短任务的瞬时 CPU 占用，而 1/2/4/8 线程
同场景测量没有显示可抵消该成本的延迟收益。识别频率负责两次业务尝试之间的等待，不能代替库内线程预算。

## 后果

未来调整默认值必须附带相同机器、相同场景的 P50/P95、CPU 和内存对照。任何模块都不得私自调用
`cv2.setNumThreads()` 覆盖统一预算；现代截图后端仍需独立评估，不能用增加匹配线程掩盖截图瓶颈。
