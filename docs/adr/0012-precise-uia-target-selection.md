# ADR 0012：精确 UIA 目标选择

- 状态：已接受
- 日期：2026-07-12

## 决策

后台单次左键的 UIA 候选优先匹配用户明确绑定的子 HWND。没有精确匹配时，依次偏好更深层、面积更小、
具有可执行动作的可见控件；离屏控件不参与选择。诊断返回扫描数、候选数、选中理由和是否命中绑定 HWND。

## 原因

仅按坐标取得第一个控件容易选中父窗口、容器或相邻覆盖控件，表面返回成功却没有触发用户想要的按钮。
稳定排序和显式 HWND 使选择结果可解释，也让真实 Win32 控件烟雾能够验证“点中了谁”而不只是“调用未报错”。

## 后果

UIA 仍只用于兼容的标准控件。超时且结果未知时不得回退 PostMessage，后台失败不得移动真实鼠标；新增
控件动作前必须保留有界 COM 线程、节点/深度/队列限制和打包后真实控件烟雾。
