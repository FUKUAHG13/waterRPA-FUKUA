# ADR 0021：可组合日志策略

日期：2026-07-13  
状态：接受

## 背景

简易、详细和完全日志原先依赖散落在执行模块中的数值判断。用户只能整体切换等级，无法只保留
识别、耗时或底层回退等正在排查的内容；继续增加日志也容易造成等级含义漂移和重复时间戳。

## 决策

1. `log_policy.py` 集中定义日志类别、中文名称和三个不可变预设。
2. 设置页保留三个预设并增加自定义模式；勾选内容后自动进入自定义，上次组合独立保存。
3. 执行调度、动作、识别、条件和主窗口事件显式标注类别，并在进入异步队列前过滤。
4. 错误、急停和严重警告属于不可关闭类别；旧的未分类调用按步骤结果处理。
5. 本地时间是一项界面显示类别，文件日志仍始终带事件发生时间。
6. 配置版本 2 保存 `log_mode` 和 `log_custom_categories`；普通方案、备份和全量包使用同一字段。

## 结果

自定义日志只增加少量纯 Python 数据与菜单项，不引入第三方依赖或明显包体积。完全与自定义调试
仍可能显著增加界面刷新；用户可关闭界面日志或只勾选需要的项目。新增日志必须选择现有类别，
确有新语义时才扩展唯一类别表并同步预设和测试。
