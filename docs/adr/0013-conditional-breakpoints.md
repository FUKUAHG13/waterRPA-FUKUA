# ADR 0013：安全条件断点

- 状态：已接受
- 日期：2026-07-12

## 决策

步骤断点可附带受限表达式。引擎在步骤执行前使用与“判断表达式”相同的 AST 白名单求值；假值继续，真值
以专用原因暂停。变量缺失或求值错误也暂停并在当前调试界面显示有界详情，避免静默漏过预期断点。

## 原因

无条件断点在长循环中需要反复人工继续，难以定位某一轮或某个变量状态。直接执行 Python 条件会扩大脚本
权限，而把错误当成假值会隐藏调试配置问题。复用安全解释器和确定赋值分析可在不增加权限的情况下定位状态。

## 后果

断点条件引用必须进入流程确定赋值分析；单步运行仍清除断点。条件、变量值和错误详情只存在于当前 UI，
不得进入运行轨迹、诊断导出或持久日志。扩展表达式语法时必须继续满足静态验证和资源上限。
