# ADR 0017：有界原生多尺度调度与安全尺度预测

- 状态：已接受
- 日期：2026-07-12

## 决策

API 10800 用一次调用共享的 1-8 线程任务池调度缩放档位、区域和结果行块，替代每个缩放档位
内部重复创建线程。自动模式按总工作量选择线程数，不设置核心亲和性；用户可选单线程或强制多核。

API 10900 的快速一个可传入多个优先尺度。DLL 在完整截图和预算检查之后先按列表匹配这些尺度，
未命中时使用相同截图与积分图处理其余尺度。全部匹配忽略优先尺度。API 10800 的单尺度入口与
旧 C ABI 入口继续保留，Python 对 API、能力位或调用异常继续回退 OpenCV。

## 原因

按尺度外层再并行会与旧的模板内线程叠加，造成过量线程和调度抖动。统一队列可以在多尺度、
多区域和单个大模板之间动态平衡工作。上次尺度是连续自动化中低成本且可验证的先验；保留完整
回退可避免把预测误差变成漏检。

## 后果

DLL 继续增加一个能力位和一个扩展函数，但不增加运行库，静态 CRT 与 Win10 x64 边界不变。快速一个
在预测命中时不再保证比较全部尺度的最高分，这与该模式既有“首个可靠目标”说明一致；全部匹配
语义不变。任务调度与并行合并必须使用确定性排序，并由真实窗口烟雾和旧 DLL 基线持续验证。
