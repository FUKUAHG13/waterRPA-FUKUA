# ADR 0003：渐进式模块化与便携多文件发布

- 状态：已接受；“仅 onedir”部分由 ADR 0016 取代
- 日期：2026-07-11

## 决策

保留 PySide6 主界面和 Python 调度器，以组合控制器和纯数据服务逐步拆分。高频执行代码按
截图匹配、条件、坐标路径和动作拆成 mixin；原生 DLL 维持稳定回退边界。发布只使用 PyInstaller
onedir，关闭 UPX，携带图标、版本信息、Python/Qt/OpenCV 和原生运行库。

## 原因

一次性重写会同时改变 UI、配置、调度和系统接口，无法可靠定位回归。多文件发布便于携带运行库、
诊断 DLL 加载并降低单文件临时解压问题。

## 后果

新底层模块不得导入主窗口。构建必须通过离线自检和干净目录冒烟测试；不得重新添加精简版、
单文件版或启动时联网行为。

后续决定恢复了同源完整版 onefile，但没有恢复精简版，也没有改变启动时不联网的约束；详见
ADR 0016。
