# ADR 0015：原生 ABI 与发行态语义对照

- 状态：已接受
- 日期：2026-07-12

## 决策

DLL API 10700 公开指针位数、跨语言结构体大小和 x64、静态 CRT、C++17、Windows 10 目标、MSVC 编译标志。
Python 加载时核对位数和结构体，发布门要求全部构建标志成立，并检查 PE 导入表没有动态 VC/UCRT 依赖。
发布目录内其余 EXE/DLL/PYD 也必须通过 AMD64 和非系统依赖闭包审计。真实屏幕烟雾同时比较原生
灰度/彩色/多尺度/全部匹配/多区域结果与 OpenCV，并由打包后 EXE 再运行一次。

## 原因

“DLL 能加载”不能发现结构体布局漂移、误打成 x86、`/MD` 运行库遗漏或坐标语义变化。这些问题往往只在
干净电脑或特定匹配模式出现。机器可读 ABI、PE 检查和同一画面的双后端对照能把风险提前到构建阶段。

## 后果

API 10700 及以上缺少 ABI 导出或结构体尺寸不一致时必须拒绝加载并回退 OpenCV。原生核心仍不接管 Python
步骤调度；修改导出结构、编译选项或匹配坐标语义时，必须同步 ctypes 声明、构建记录、诊断和发行态烟雾。
