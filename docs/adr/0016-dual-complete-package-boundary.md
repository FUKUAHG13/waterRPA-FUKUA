# ADR 0016：同源完整版双构建边界

- 状态：已接受
- 日期：2026-07-12

## 决策

同一套 `fukua_rpa` 完整版源码同时维护 PyInstaller onedir 与 onefile。两种产物共享产品版本、
配置格式、功能、图标、Python/Qt/OpenCV 运行库、可选原生 DLL 和发行态自检；不建立两套业务源码。
默认发布脚本只构建推荐的 onedir。只有用户明确要求单文件版或两种格式时，才使用
`--format onefile` 或 `--format all`；这项默认值也写入根目录 `AGENTS.md`，防止后续维护时误打包。

多文件版是推荐产物，继续生成构建记录、SBOM、许可证、真实签名状态、运行时闭包、包内哈希、
外部逐文件清单和便携 ZIP。单文件版是便携产物，必须核对 AMD64 启动器，并从冻结 EXE 直接运行
环境自检、UIA 烟雾和原生/OpenCV 对照烟雾；自检还要确认配置等可写数据仍落在 EXE 所在目录，
而不是 PyInstaller 临时解压目录。两种规格都关闭 UPX。

精简版保持终止，不进入构建脚本、测试矩阵或独立源码分支。程序启动时仍不联网。

## 原因

单文件版方便普通用户传输和直接启动，多文件版则启动更快、误报风险更低且更容易核验依赖。
它们只是冻结与分发方式不同，不应产生功能分叉。针对各自结构使用不同的发行校验，能在保留
便携性的同时避免假设 onefile 临时解压内容可像 onedir 一样逐文件审计。

## 后果

可选发布测试矩阵增加一个冻结格式，构建时间和磁盘占用上升。onefile 冷启动更慢，也可能更容易被
杀毒软件启发式拦截，因此文档必须把 onedir 标为推荐版。任何功能变更只修改共享源码；若某功能
只能在一种冻结格式运行，必须先修复路径或资源边界，不能复制业务实现。
