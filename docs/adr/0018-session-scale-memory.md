# ADR 0018：会话内多倍率记忆与完整回退

- 状态：已接受；记忆身份规则由 ADR 0020 修订
- 日期：2026-07-12

## 决策

“快速一个”按模板绝对路径、文件代次和灰度模式维护进程内有界命中历史。缩放网格是当前可用
范围而不是模板身份，范围变化会过滤已不再可用的历史倍率，但不会创建同名新记录。排序同时考虑命中
频率、近期程度和匹配质量；用户填写的手动优先倍率先于自动学习倍率。稳妥、均衡、积极只调整
算法覆盖率和容量倾向，不映射为固定数量。只有用户启用自定义容量后，学习倍率上限和历史次数
上限才覆盖算法结果。

API 10900 通过 `wrpa_find_template_ex2` 一次接收优先倍率列表。优先列表全部未命中时，必须复用
相同截图与积分图搜索全部剩余倍率。API 10800 单倍率入口、旧基础入口和 OpenCV/Python 回退继续
保留；OpenCV 路径使用同一优先顺序。全部匹配始终搜索完整倍率集合。

自动学习记录不写入磁盘，不进入方案、备份、剪贴板或全量包，关闭软件即清空。手动优先倍率
属于普通用户配置。设置页显示每张模板当前优先倍率与动态历史上限；完全日志只在优先集合或
容量变化时记录，详细/完全点击日志显示本次实际命中倍率。

## 原因

单个“上次倍率”无法覆盖几个尺寸轮流出现的页面，永久学习又会引入过时状态、隐私和迁移问题。
会话内有界统计能利用当前任务的真实分布，同时容易清除、容易解释。完整回退保证记忆只改变
搜索顺序，不改变可识别范围，也不需要机器学习或新增大型运行库。

## 后果

每个进程最多维护有限模板条目和有限历史，访问受锁保护；模板文件代次变化会在同一身份槽位清空
旧历史，缩放网格变化只保留交集。手动值不在当前网格时忽略，不做近似扩大。任何未来跨重启持久化都必须另立
决策，明确版本、失效、隐私和容量边界，不能直接序列化当前内存结构。
