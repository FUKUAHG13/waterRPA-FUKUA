# 领域文档

本项目采用单一上下文文档结构。修改代码前，按需读取根目录的 `CONTEXT.md` 和相关的 `docs/adr/`。

## 读取规则

- `CONTEXT.md` 用于记录项目术语、核心概念及各概念之间的关系。
- `docs/adr/` 用于记录已经确定的架构决策及其原因。
- 这些文件不存在时直接继续，不主动创建空文档。只有真正确定了术语或架构决策后才创建。

## 术语规则

代码、测试、任务和文档应使用 `CONTEXT.md` 中定义的统一术语，不随意创造同义名称。

需要的概念尚未记录时，应先判断它是项目确实缺少的术语，还是不必要的新名称。

## 架构决策冲突

若计划中的修改与现有 ADR 冲突，必须明确指出冲突的 ADR 和原因，不得静默覆盖已有决策。
