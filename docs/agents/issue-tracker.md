# 任务管理：本地 Markdown

本项目的需求规格和实施任务使用仓库内的 `.scratch/` 目录管理，不依赖外部任务平台。

## 文件约定

- 每个功能使用独立目录：`.scratch/<功能名>/`。
- 需求规格保存为 `.scratch/<功能名>/spec.md`。
- 子任务分别保存为 `.scratch/<功能名>/issues/<编号>-<任务名>.md`，编号从 `01` 开始。
- 不把多个独立任务合并到同一个任务文件。
- 对话、调查结果和补充信息追加到任务文件的 `## Comments` 部分。

## 技能操作约定

- 技能要求“发布任务”时，在对应的 `.scratch/<功能名>/` 目录中创建文件。
- 技能要求“读取任务”时，读取用户指定的任务路径或编号。
- 创建目录和文件前先检查是否已有同名任务，避免覆盖现有记录。
