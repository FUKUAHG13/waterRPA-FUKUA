## 这是什么

浮夸改 RPA 是一款面向普通电脑用户的 Windows 图形化自动化工具。你不需要编写 Python、脚本语言或命令行代码，只需在界面中依次添加“点击、按键、等待、识别图片、判断条件”等步骤，就能组合出自己的自动操作流程。

它既可以按照固定坐标操作，也可以先在屏幕上寻找目标图片，再点击目标；因此即使按钮位置发生小幅变化，流程仍有机会继续工作。适合处理重复点击、固定界面流程、周期检查、软件操作复现和简单桌面测试等任务。

## 核心优势

### 不懂编程也能搭建流程

- 使用图形界面添加、复制、排序和配置步骤。
- 支持点击、双击、拖拽、悬停、滚轮、系统按键、等待、截图和提醒等常用操作。
- 每一步都可以单独设置重复、失败处理、超时、跳转和生效范围。

### 不只会点固定坐标

- 支持 PNG、JPG、BMP 图片识别，找到目标后再执行点击。
- 支持缩放、灰度、相似度和图片内相对点击位置。
- 可限制全局或单步骤识别区域，也可组合多个相隔较远的小区域，减少无效扫描。
- 同时出现多个目标时，可选择只点一个或全部点击，并按位置、距离等方式安排顺序。

### 能表达更完整的自动化逻辑

- 支持有限次数、指定时间和持续循环。
- 支持成功或失败后跳过步骤、跳至指定步骤，以及连续失败后的处理方式。
- “直到条件成立”可等待图片出现、图片消失、区域变化或区域变成指定状态，再继续后续流程。
- 支持超时跳过与超时急停，避免流程在异常状态下无限运行。

### 点位操作直观可检查

- 坐标可直接取点，不必手工抄写屏幕位置。
- 支持自定义点位、等距步进、起点与终点路径和多点连续操作。
- 可一次预览脚本中的坐标点击位置，提前发现点错、漏点或顺序问题。
- 预览中可拖动修正点位，复杂流程也更容易校准。

### 快捷键也能变成自动操作

- 可设置全局启动与停止热键。
- 按键映射可以把字母、数字、功能键或组合键绑定为指定位置的点击。
- 可选择真实鼠标点击、点击后返回原位，以及不移动鼠标的后台窗口点击实验模式。

### 方案可以带走

- 支持多个方案，常用流程可以分别保存和切换。
- 普通导出用于保存步骤配置。
- 全量导出可连同方案引用的图片一起打包，换到另一台电脑后不必重新整理相同路径。

### 兼顾速度与兼容性

- 可选原生图片识别核心，用于降低部分高频识别场景的延迟。
- 原生核心不可用或不适合当前任务时，会自动回退到 Python/OpenCV 识别路径。
- 支持识别频率、自适应降频、限定识别区域等方式，避免无意义地持续占满 CPU。

## 三步开始使用

1. 前往 [Releases](https://github.com/FUKUAHG13/waterRPA-FUKUA/releases) 下载最新完整版。
2. 解压多文件版并运行文件夹内的 EXE，或直接运行单文件版。
3. 在主界面添加步骤、填写坐标或选择图片，设置启动和停止热键后开始运行。

多文件版启动更快，也更容易检查文件是否完整，推荐日常使用；单文件版更方便携带，但启动时需要释放内置运行文件，因此通常稍慢。两种完整版都已包含所需运行环境，新电脑无需另行安装 Python 或 VC++ 运行库。

## 适用范围

- Windows 10 1809 及以上版本，仅支持 x64。
- Windows 11 x64。
- 主要面向能够在 Windows 桌面上看见并操作的普通软件界面。

浏览器画布、游戏、DirectX、自绘控件、高权限窗口或具有特殊输入保护的软件，可能无法使用后台点击；遇到这种情况可改用真实鼠标点击，或尝试以管理员身份运行。图片识别效果也会受到系统缩放、界面主题、动画和截图质量影响，建议截取尺寸适中、特征清晰的目标图片。

## 隐私与联网

软件的方案、图片和运行日志默认保存在本地。程序启动时不会主动联网；设置中的作者主页与 GitHub 按钮只会在用户主动点击后打开对应网页。
